引用作为函数参数时，引用传递方式是将形参定义成对应的实参的引用，即形参作为对应实参的别名 1、引用必须初始化，但是不分配存储空间。指针在定义时不一定初始化，定义指针需要分配存储空间。

2、引用初始化之后不能被改变，指针可以改变

3、不存在指向空值的引用，但存在指向空值的指针

类和对象

类的定义

  • 类成员访问权限 private声明私有成员，私有数据成员只允许在类内访问，私有函数只允许被类内的其他成员函数调用

public声明公有成员，如上 允许、允许

protected声明保护成员。如上 允许类内或者子类访问

访问权限的好处：信息隐蔽，实现封装/数据保护，防止被修改。

默认情况下是private

可以以任意的顺序进行声明公私有成员

  • 内联函数 内联函数：类的声明体中实现的函数，类的声明体外使用inline表示的函数。

编译器遇到调用内敛函数的地方会用函数体的代码代替函数的调用，好处是节省函数调用带来的参数传递、栈空间的开销，从而提高执行速度，代价是增加了代码的长度

内联函数注意事项： 1、使用inline关键字的函数不一定会被编译器调用时展开，inline对编译器只是建议作用。

2、通常只讲较短的函数设计为内联函数。

3、内联函数不允许有循环和开关语句，如果有则视为普通函数

4、递归函数不能用作内联函数，因为无法在调用点完全展开(虽然第一次可以)

5、定义必须出现在第一次调用之前

内联函数和宏定义

二者可以达到同样的目的，但是区别如下：

1、内联函数在执行时可以调试，而宏定义不可以

2、编译器会对内联函数参数做安全检查或者自动类型转换

3、内联函数可以访问类的成员变量

4、类中声明同时定义的成员函数自动转换为内联函数

  • 构造函数 构造函数是特殊的成员函数，主要用于对成员进行初始化

构造函数名称和类名相同

  • 类对象

类对象的存储方式：

每个对象单独存储，按照非静态数据成员定义的先后顺序依次存放，成员函数放在代码区，所有对象共享，不占对象的内存空间。调用时用对应成员函数的地址代替。

类对象的大小：所有非静态成员的类型大小之和，普通成员函数与sizeof()无关。

有虚函数时要有一个虚函数表指针，另外存在虚继承时还需要指向虚基类的指针，每个指针占用一个地址大小的内存空间(4个字节)

  • 空类

为什么sizeof(空类)=1? 实例化的原因,每个实例应该有唯一的地址,为了达到这样的目的，编译器会给空类插入一个字节，使空类在实例化后在内存中得到的唯一地址 空类所占内存为1字节

  • 类与结构体的区别 实际上类是从结构体演化过来的，但是对结构体进行了扩展。

默认成员 结构体是公有的，类是私有的

提出静态数据成员目的是解决数据共享的问题

友元和运算符重载

友元函数

  • 友元函数概述

friend 函数类型 友元函数名(参数表)

指虽然不是类成员函数却能够通过类对象访问其所有成员函数。类授予他的友元特别的访问权

特点：

1、不是类的成员函数，但需要在类中声明

2、友元函数不能直接访问类的成员，只能通过类的对象访问其所有成员，包括私有和保护的成员

3、不是类的成员函数，没有this指针

4、友元函数的调用方式和普通函数调用方式相同，不需要通过对象调用

5、友元函数可以作为另一个类的成员函数，这种情况下他既有友元函数的特点，又有成员函数的特点

6、友元函数通过对象访问其所有成员，破坏了数据的安全性，所以使用友元函数必须非常谨慎，不要通过友元函数对数据成员进行危险操作。

定义友元函数的说明：

1、本质上友元函数是全局的，声明位置可以放在类的任何地方，放在public、private、protected意义完全一样。

2、原则上讲友元函数在类里声明，在类外实现。在类内实现的友元函数在调用时必须具有该类类型的实参

  • 操作多个对象数据成员的友元函数

一个函数需要操作多个类对象的非公有成员，可以将其设计成这些类的友元函数，并在这些类中分别声明。

1、将这个函数声明声明为所有类的友元函数。

2、函数声明为一个类的成员函数、其他一个或者多个类的友元函数

如果一个类的成员函数是另一个类的友元函数，则成这个成员函数为友元成员。通过友元成员不仅能访问自己所在的类对象中的所有其他成员，还可以访问有关键字friend声明语句所在类的对象中的所有成员。

注意：如果存在友元成员，必须先定义成员函数所在类，声明友元函数时要加上成员函数所在类的类名和作用域运算符。

友元类

可以将一个类声明为另一个类的友元类。若B是A的友元类，则B的所有成员函数都是A的友元函数。

class A{
    friend class B;
}

运算符重载

  • 概念 运算符重载是用同一个运算符完成不同的运算功能，和函数重载一样，在编译阶段完成，体现的是静态的多态性。

C++运算符重载的相关规定如下：

1、不能改变原运算符的优先级

2、不能改变原运算符的结合性

3、默认参数不能和运算符重载一起使用，也就是说在设计运算符重载函数时不能使用默认参数

4、不能改变运算符的操作数数量

5、不能创建新的运算符，只能在现有运算符上重载

6、运算符作为c++内部提供的数据类型时，原来含义不变

7、运算符可以被重载用于用户定义的类对象或者用户定义的类对象与内置数据类型变量的组合

虚函数和多态性

虚函数

  • 虚函数的声明 函数绑定是将函数入口和函数调用相联系的过程，分为静态绑定和动态绑定

静态绑定在程序执行前完成，由编译系统或者操作系统装入程序计算函数的入口地址。

动态绑定(迟后编译)在程序执行过程中完成，由程序自身计算函数入口地址。

  • 设计虚函数的注意事项

  • dynamic_cast运算符

  • 带虚函数的派生类对象的内存结构

当一个类中含有虚函数时，其对象存储在非静态数据成员最前面添加虚函数指针vfptr指向虚函数表。

虚函数表中存储着当前类对象的所有虚函数地址，摆脱类型制约

纯虚函数和抽象类

  • 纯虚函数 特殊的虚函数，被标明为不具体实现的虚函数

语法上，纯虚函数是在虚函数后面加上 =0，表示该虚函数无函数体，此处=不是赋值运算

基类中不能对虚函数给出有意义的实现，实现留给派生类去做。

含有纯虚函数的类称为抽象类

  • 抽象类的特点

1、只能用作其他类的基类，不能建立抽象类对象

2、抽象类不能用作参数类型、函数返回类型和显示转换类型

3、可以定义和指向抽象类的指针和引用，辞职真可以指向他的派生类，进而实现多态性。与抽象类相对应，能够建立对象的类叫具体类
