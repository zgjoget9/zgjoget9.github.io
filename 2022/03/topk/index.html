TopK

March 16, 2022

最近在刷面经，经常看到遇到TopK的问题。现在总结一下

TopK是在一堆数据中找出最大/小的K个数

完整快排

在数据量不大的情况下，直接快排整个数据，时间复杂度为O(nlogn)

冒泡排序

我们都知道冒泡排序每次都会选出一个最值就序，所以只需要进行K次冒泡就能够完成

由于需要遍历K次数据，时间复杂度为O(K*n)

部分快排

快排是选出一个index之后，以index为基准，index左右边分别为大于/小于index的值，进而对index左右边进行递归，类似于非严格二分查找。

时间复杂度为O(n + n/2 + n/4 +… < 2n)=O(n)

缺点：数据量过大可能无法一次性将数据全部加载入内存。

堆

由于我们只是想得到TopK，并不要求TopK有序，所以可以建立大/小顶堆，然后遍历一次整个数据，将每个数据与堆顶数据比较，对应的进行对元素的替换，替换后重新建立顶堆。

维护堆的使用红黑树，最坏的时间复杂度为O(logn),遍历一遍数据为O(K)

总时间复杂度为O(Klogn) 代码实现

vector<int> topK(vector<int>nums,int k) {
    make_heap(nums.begin(), nums.begin() + k);
    for (int i = k; i < nums.size(); i++) {
        if (nums[i] < nums[0]) {
            swap(nums[i], nums[0]);
            make_heap(nums.begin(), nums.begin() + k);
        }
        else continue;
    }
    vector<int>res(nums.begin(), nums.begin() + k);
    return res;
}

物理层面

分布式思想

将数据分散在多台机器中，每台机器处理自己的顶堆，最后将各自顶堆合并，继续处理。
