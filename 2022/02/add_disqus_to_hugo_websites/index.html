Add_disqus_to_hugo_websites

February 1, 2022 in website, hugo

写了几篇博客之后发现底下有个We were unable to load Disqus，然后由于好奇就去一探究竟….

  • 问题：

本网站使用hugo搭建，我在config.toml中字符搜索disqus，找到如下两处：

然后通过搜索引擎了解到disqus其实是一个评论系统，由于我是直接clone了hugo-tranquilpeak-theme后面跟着guide修改信息之后，就没怎么管，这几天摸鱼的时候想着要把这个问题解决掉，所以我首先去注册了一个disqus账号，然后去查看了disqus的官方文档

发现这种问题一般是自己网站的域名和配置的shortname不匹配造成的。 而我注册disqus还没有添加网站，所以还没有shortname,disqus自然就无法加载。 首先我添加了我的网站就有了disqusshortname，但是在install的模板中又没有发现hugo….所以只能使用universal code来配置。
于是我将上面在toml文件中检索到的disqus的shortname修改为自己disqus中显示的shortname,然后rebuild，deploy发现还是无法加载disqus。 所以这里我卡住了….

然后这里我又在disqus官网探索了一番，看完了一个全英文的universal code配置视频演示后，我使用搜索引擎搜索hugo and disqus，发现有几个前辈搭建的hugo网站也是无法加载disqus的，仔细翻阅后，参考他们的解决办法，最终得以解决，参考链接

  • 解决方法：

universal code的位置：https://disqus.com/admin/install/platforms/universalcode/

universal code如下：

<div id="disqus_thread"></div>
<script>
    /**
    *  RECOMMENDED CONFIGURATION VARIABLES: EDIT AND UNCOMMENT THE SECTION BELOW TO INSERT DYNAMIC VALUES FROM YOUR PLATFORM OR CMS.
    *  LEARN WHY DEFINING THESE VARIABLES IS IMPORTANT: https://disqus.com/admin/universalcode/#configuration-variables    */
    /*
    var disqus_config = function () {
    this.page.url = PAGE_URL;  // Replace PAGE_URL with your page's canonical URL variable
    this.page.identifier = PAGE_IDENTIFIER; // Replace PAGE_IDENTIFIER with your page's unique identifier variable
    };
    */
    (function() { // DON'T EDIT BELOW THIS LINE
    var d = document, s = d.createElement('script');
    s.src = 'https://EXAMPLE.disqus.com/embed.js';
    s.setAttribute('data-timestamp', +new Date());
    (d.head || d.body).appendChild(s);
    })();
</script>
<noscript>Please enable JavaScript to view the <a href="https://disqus.com/?ref_noscript">comments powered by Disqus.</a></noscript>

我们需要根据自己的网站修改s.src的内容。

最后将disqus的universal代码保存在新文件comments.html文件中。

将这个comments.html放到网站的header.html的文件夹中。这里我发现在theme的文件夹里有两个地方有header.html,所以我在两个地方都放了comments.html

最后rebuild and deploy，完美解决！效果如下（国内可能无法访问）：
